I found a very high-quality video on YouTube made for the same unit. I clicked through because I genuinely wanted to know how someone had made it that well. Later I realised the student had already made short films and edited videos at this level before.

My first reaction was: so this training was already there. It was an existing strength that suddenly showed up in an IT assignment. That felt pretty amazing.

Visual education, editing, and storytelling are not irrelevant extras beside technical work. Language, shot selection, interaction rhythm, and information hierarchy all shape how a project is understood. Skills that usually stay invisible can suddenly appear together in a project.

It also made me feel that a technical project is not only evidence that someone can code. A project does not need a huge scope. Completing one clear experience often reveals more about judgement and long-term accumulation.
