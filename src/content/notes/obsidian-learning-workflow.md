虽然 24 年就知道 Obsidian，但实在话它很像毛坯，需要自己配置插件，刚开始有点麻烦。附件管理插件我个人感觉非常必要，不然插入图片多了会很混乱；云端同步也得自己选方案。移动端我会精简一点，Mac 里插件多一点。

之前还是用 OneNote 多。后来考虑到笔记越来越精简，我也爱用 AI 整理东西，这段时间就转用 Obsidian。直接 Copy/Paste AI 输出，Markdown 文件渲染以后阅读很舒服，导出 PDF 也不会像传统笔记软件那样容易跑版。

缺点是自己要配置，优点也是可玩性高。边编辑边预览可能有点极客，知识图谱也需要慢慢驯化，不过我个人还是很喜欢。

它的插件基本都是开源的，真的是很好的一群人。附件、目录、同步和知识图谱都可以按自己的习惯慢慢补，不需要一开始就配成一套很重的系统。

Markdown 其实很常见，只是大众不一定知道。最常见的例子就是 AI 输出里的 `**粗体**`。直接复制成纯文本看起来可能不美观，放进 Obsidian 里就很舒服。

在允许使用 AI 的学习场景里，我会让工具读取指定的本地 Folder、提取课程内容，再整理成 Markdown、清单或 Mermaid 图。人的工作还是限定范围、核对来源、补充重点，不会把未经检查的输出直接当成答案。

也有人直接把 Obsidian 连到 Claude Code、Cursor 或类似 VS Code Copilot 的工具。如果有长期写日记或积累笔记的习惯，这类 Agent 或 LLM 确实可能逐渐理解你的资料。不过我更愿意先把它当成本地文件上的检索和整理工具，而不是夸张成“最了解你的人”。

我从去年开始也一直在迭代类似的玩具，参考过 YouMind，再往 Agent 和本地知识工作流的方向试。

Mac 的话还可以推一下 Arc 浏览器，管理网页和沉浸浏览都很爽。我用了挺久，偶尔有 Bug 的时候再用 Chrome 就行。
