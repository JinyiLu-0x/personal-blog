当时课程要求和成绩计算散落在很多地方，老师和 Tutor 的说明也比较碎。光是把作业要求、时间线和成绩计算理清，就要花很多时间。

所以我花了几个小时 Vibe 了一个 Todo List 网站，完全按照自己的喜好来。技术栈主要是 React，没有什么特别复杂的东西。

第一版只做了 Web 前端，数据放在浏览器本地存储里。关掉页面记录还在，只要没有清理浏览器缓存。后端当时先留着，重点是先把自己的实际问题解决掉。

这个项目比较能体现我怎么用技术改善工作流：

1. 先从被允许使用的资料里整理日期、要求和依赖关系；
2. 把重复查看和手动计算变成结构化界面；
3. 用 AI 加速原型、排版和调试；
4. 自己核对课程原始材料，并决定最终记录内容。

用起来感觉挺顺手，我也把自己的 SemTrack Demo 公开了。这个项目更重要的意义，是体现我怎么把实际问题整理成产品，并持续优化自己的工作流。它不是为了替人完成作业，而是减少整理课程信息和反复查找的时间。
