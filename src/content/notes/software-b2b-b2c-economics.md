最开始看到一句话：软件行业做 ToB 合适，因为卖一个和卖一千个花的时间差不多；但实体卖一个和卖一千个，投入的物流、仓储和库存都不一样。

我当时觉得这个说法很好理解，甚至拿它解释为什么 ToC 用户更多的 ChatGPT，会被更重视 ToB 和开发者场景的 Claude 追上，尤其是 Claude Code 出现以后。ChatGPT 的大众认知当然更强，但“大家都听过”和“产品能不能从用户身上赚到钱”不是一件事。

后来又发现 AI 跟传统 SaaS 还是不完全一样。软件复制一份的成本很低，但 AI 每次调用都真的要消耗算力。用户越多不一定只是摊薄固定成本，也可能一起放大推理成本。所以这里不能直接套“卖一份和卖一千份都一样”。

还有使用习惯的问题。我跟一位年长的经营者聊过，他明显感觉身边开始有更多人使用某个大众 AI 产品，也认为这群人存在付费市场。我一开始不认同，因为觉得免费用户多、模型本身也没有强到必须付费；但他说习惯很难改变，就像大家为什么一直用同一个社交软件。这个修正我觉得有道理：模型能力不是唯一变量，使用习惯也可能形成留存。

后续补充了几个修正：

- C 端存在竞品并不意味着没有机会，产品可以在成熟范式上继续改进；
- C 端用户会形成情感、品牌和使用习惯，这些都可能构成留存；
- B 端客户不一定更忠诚，他们会持续评估价格，并尝试获取更多利润空间；
- B 端购买决策可能只需要说服少数人，但销售周期、集成成本和组织风险也更高；
- 简单易用通常比复杂功能更重要，但仍要建立可信度和清晰的市场进入路径。

我当时还想到过几个不完全严谨、但有助于理解的例子。DeepSeek 的低价 Token 更像是用别的业务逻辑补贴使用成本；YouMind 看起来像面向文字工作者的知识工具，但我一直不确定它到底更像 ToB 还是 ToC。推荐给某个老板使用，并不自动等于企业采购，最后也可能还是个人在用。

这也和我之前记下来的增长问题连起来了：同样是 ToC，推荐、留存和利润转化的能力不同，能承受的拉新成本就完全不一样。相关记录在这里：[增长与商业化其实是同一套单位经济](/notes/growth-and-monetization)。

聊天里还提到 SaaS Sales：B 端可能只需要说服一个人或少数几个人，但这不代表它天然更容易。客户也会压价格、要求集成和售后，销售与交付成本不能当作不存在。

所以现在我不会再直接说“软件行业不能做 ToC”或者“ToB 一定更好”。它们是两套不同的成本、购买决策和留存结构。Ody 后来公开分享的一些修正我也挺认可，所以把原链接留在这里；正文已经能说明我的逻辑，就不再堆截图当 supporting evidence。
