差异化要做“止痛药”，不是维生素。

技术人很容易先做一个通用工具，再让用户自己想怎么用。但如果问题不痛，功能再多也只是新鲜一阵。

另一个很具体的判断是：当产品需要用户自己填 API Key 时，就已经主动放弃了大量普通用户。早期开发者产品这样做很常见，但真正面向大众时，用户不应该先理解模型供应商、额度和密钥管理，才能开始完成自己的任务。

这个问题放在中国市场会更明显。用户可能还要处理模型平台注册、实名认证、支付方式、网络环境、余额、额度和不同供应商之间的切换。产品如果把这些步骤全部交给用户，实际上不是“给用户更多自由”，而是把集成成本重新推回去了。

比较合理的产品结构应该允许用户自己选择：

- 默认使用产品内置的模型与额度，打开就能用；
- 高级用户可以切换供应商或填写自己的 API Key；
- 对隐私或成本敏感的人，可以选择兼容的本地模型；
- 产品明确展示成本、能力和数据去向，而不是把选择藏起来。

细分 AI 产品真正要判断的是：

- 用户是否真的愿意为结果付费；
- 场景是否足够具体；
- 产品能否主动完成任务，而不只是多一个“创建任务”的按钮；
- 是否把模型、API 和工程复杂度藏在产品后面。

所以“Build an Agent, not a Tool”对我来说，不是凡事都加 Agent，而是别把所有流程重新丢回给用户。
