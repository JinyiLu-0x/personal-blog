**Prompt → Context → Harness**

I vaguely remember Context becoming a popular idea around July or August last year. Looking back at this pace of change, my own Vibe Coding path is pretty interesting:

- In 2024 I copied and pasted directly from ChatGPT into VS Code. Debugging was bad, which was really also a context problem.
- Then I moved to IDEs such as Cursor and used Copilot. The model was no longer guessing through a separate chat window.
- Around mid-2025, Claude moved from plain chat to “chat on the left, webpage on the right,” with one-click deployment and project chats. The first time I saw it, I was genuinely amazed.
- By late 2025, Google AI Studio followed a similar pattern, with a major Gemini update and GitHub push support, though it was still browser-based.
- Then came Antigravity, back inside an IDE-like environment. I used it more when other quotas shrank, but it sometimes hallucinated and changed things randomly, so the Diff still needed watching.
- In early 2026 I started using Codex. Honestly, it was impressive: the integration of a project chat, one-click GitHub push, a clean separation between chat and IDE, and an easy jump into VS Code for detailed edits. It is currently the smoothest fit for me.

That is a small personal summary of my Vibe Coding experience. Tools have slowly moved from “give me code” to “understand the project, use tools, complete work, and let me confirm it.”

Context Engineering was not a temporary buzzword. It became a foundation for later Harness and Agent workflows.
